# htbox CLI 工具设计文档 (Rust 版)

> 日期: 2026-04-13
> 状态: 草稿
> 语言: Rust

## 1 项目概述

**项目名称**: htbox  
**项目类型**: Linux CLI 工具  
**核心功能**:

1. **系统服务管理** - 将常用命令或脚本注册为服务，支持常驻服务和开机执行一次的服务
2. **快捷命令管理** - 将常用命令或运维脚本统一管理，支持别名、注释、模板参数化

### 技术栈

| 组件 | 选择 | 版本 |
|------|------|------|
| CLI 框架 | clap | ^5.0 |
| 异步运行时 | tokio | ^1.0 |
| 配置解析 | toml | ^0.8 |
| 日志 | tracing | ^0.1 |
| 错误处理 | thiserror + anyhow | latest |

## 2 CLI 命令结构

```
htbox service <command>     # 服务管理
  start <name>               # 立即启动服务进程
  stop <name>                # 停止服务进程
  restart <name>             # 重启服务
  enable <name>              # 设置开机自启
  disable <name>             # 取消开机自启
  status <name>              # 查看状态(+日志+资源)
  list                       # 列出所有服务(显示enable状态)
  add                        # 添加服务(交互式/参数)
  edit <name>                # 编辑服务配置
  remove <name>              # 删除服务（先停止，再清理配置）
  logs <name>                # 查看服务日志
    [-n <lines>]             # 查看最近N行
    [-f]                     # 实时跟踪
    [--errors]               # 只看错误日志
  env <subcommand>           # 服务环境变量管理
    list <name>              # 列出服务环境变量
    add <name> KEY=VALUE     # 添加服务环境变量
    remove <name> KEY        # 删除服务环境变量

htbox cmd <command>          # 快捷命令管理
  list                       # 列出所有命令
  add <name>                 # 添加命令(交互式)
  run <name> [args...]      # 执行命令
  edit <name>               # 编辑命令
  remove <name>              # 删除命令

htbox --help                 # 帮助信息
htbox --version              # 版本信息
```

### 语义说明

| 操作 | 说明 |
|------|------|
| `start` | 立即启动服务进程（相当于 systemctl start） |
| `enable` | 设置开机自启（相当于 systemctl enable） |
| `list` | 列出服务，显示 Running + Enabled 状态 |

## 4 服务配置格式

### 4.1 目录结构

每个服务一个独立目录：

```
~/.htbox/services/<service-name>/
├── service.toml           # 服务配置
├── wrapper.sh            # Wrapper 脚本（自动生成，加载环境变量后 exec runner.sh）
├── runner.sh             # 业务脚本（用户编辑）
├── .env                   # 服务专属环境变量
├── run/                   # 运行时目录（PID 文件）
└── logs/                  # 日志目录
    ├── stdout.log
    └── stderr.log
```

> **注意**：wrapper.sh 和 runner.sh 由 htbox 自动生成。用户编辑 runner.sh 添加业务逻辑。

### 4.2 service.toml 格式

```toml
# 服务基础信息
name = "my-service"
description = "这是一个数据同步服务"

# 服务类型
type = "daemon"           # daemon=常驻服务 | onetime=系统启动执行一次

# 脚本路径（相对于当前目录，或绝对路径）
script = "script.sh"

# 启动行为
start = "immediate"        # immediate=立即启动 | manual=手动启动
restart_policy = "on-failure"  # on-failure | always | never
restart_delay = 5          # 重启延迟秒数

# 开机自启配置
auto_start = true

# 运行用户（可选，默认当前用户）
# user = "ubuntu"

# 环境变量 .env 文件（可选，默认使用同目录下的 .env）
# env_file = ".env"

# 日志配置（可选）
[logging]
  stdout = "logs/stdout.log"
  stderr = "logs/stderr.log"
  max_size = "10M"        # 单文件最大10MB
  max_files = 5           # 保留5个历史文件
  compress = true         # 压缩历史日志
```

### 4.3 脚本模板 (runner.sh)

> **注意**：脚本模板在**创建服务时渲染**，生成实际的脚本文件。实际运行时使用 wrapper.sh 加载环境变量后执行 runner.sh。

#### runner.sh (业务脚本，用户编辑)

```bash
#!/bin/bash
# ==========================================
# my-service - 数据同步服务
# Generated by htbox
# ==========================================

set -e

# ========== 在此添加你的业务逻辑 ==========

echo "[$(date)] my-service started"

# 示例：保持运行的进程
# exec /your/command/here

# 示例：一次性脚本
# echo "Done"
# exit 0
```

#### wrapper.sh (自动生成，勿编辑)

```bash
#!/bin/bash
# ==========================================
# my-service - Wrapper Script
# Generated by htbox - Do not edit
# ==========================================

set -e

# Load service environment variables
SERVICE_DIR="$(cd "$(dirname "$0")" && pwd)"
if [ -f "$SERVICE_DIR/.env" ]; then
    source "$SERVICE_DIR/.env"
fi

# Load global environment variables
if [ -f "$HOME/.htbox/envs/global.env" ]; then
    source "$HOME/.htbox/envs/global.env"
fi

# Execute the actual service script
exec "$SERVICE_DIR/runner.sh" "$@"
```

### 4.4 .env 模板

服务专属环境变量文件，格式与 systemd EnvironmentFile 一致：

```bash
# my-service 环境变量
# 在此定义服务专属的环境变量

# 示例
DATABASE_URL=postgres://localhost:5432/mydb
REDIS_URL=redis://localhost:6379
```

## 5 快捷命令配置格式

### 5.1 命令配置 (TOML)

```toml
# commands/deploy.toml
name = "deploy"
description = "部署应用到K8s集群"

# 模板命令，支持参数化（支持多行）
command = """
kubectl config use-context {{.cluster}} &&
kubectl apply -f {{.file}} &&
echo "Deployment completed at $(date)"
"""

# 超时时间（秒）
timeout = 300

# 参数定义
[params]
  cluster = { required = true, default = "prod", description = "集群名称" }
  file = { required = true, description = "部署文件路径" }
  dry_run = { required = false, default = "false", description = "试运行模式" }

# 使用示例
[examples]
  prod = "htbox cmd run deploy --cluster=prod --file=prod.yaml"
  dev = "htbox cmd run deploy --cluster=dev --file=dev.yaml --dry-run=true"
```

### 5.2 模板引擎

使用 Rust 的 `tera` 模板引擎，支持：
- `{{key}}` 变量替换
- 条件 `{% if flag %}...{% endif %}`
- 循环 `{% for item in items %}...{% endfor %}`

#### 5.3 命令行参数解析

快捷命令的参数通过命令行传递：

```bash
# 格式：--key=value 或 --key value
htbox cmd run deploy --cluster=prod --file=prod.yaml

# 转换为模板变量
{
  "cluster": "prod",
  "file": "prod.yaml"
}
```

**解析规则**：
- `--key=value` → `key=value`
- `--key value` → `key=value`
- 短横线转下划线：`--dry-run` → `dry_run`
- 布尔类型：`--verbose` → `verbose=true`

**参数校验**：
- `required=true` 的参数必须提供，否则报错
- `required=false` 但未提供时使用 `default` 值

## 7 后端实现策略

### 7.1 后端检测逻辑

```rust
use std::path::Path;
use std::process::Command;

fn detect_backend() -> Backend {
    // 1. 检查 /run/systemd/system 是否存在（基础检查）
    if !Path::new("/run/systemd/system").exists() {
        return Backend::Cron;
    }

    // 2. 尝试执行 systemctl --version（验证命令可用）
    if Command::new("systemctl").arg("--version").output().is_err() {
        return Backend::Cron;
    }

    // 3. 尝试执行 systemctl list-units（验证systemd运行正常）
    let output = Command::new("systemctl")
        .args(["list-units", "--type=service", "--state=running", "--no-pager"])
        .output();
    
    match output {
        Ok(out) if out.status.success() && !out.stdout.is_empty() => {}
        _ => return Backend::Cron,
    }

    // 4. 检测容器环境
    if is_container_environment() {
        return Backend::Cron;
    }

    Backend::Systemd
}

fn is_container_environment() -> bool {
    // 检查 /proc/1/cgroup
    if let Ok(content) = std::fs::read_to_string("/proc/1/cgroup") {
        let indicators = ["/docker/", "/containerd/", "/kubepods/", "containerd"];
        if indicators.iter().any(|i| content.contains(i)) {
            return true;
        }
    }

    // 检查 /.dockerenv
    Path::new("/.dockerenv").exists()
}
```

### 7.2 后端选择规则

| 场景 | 检测结果 | 使用后端 |
|------|----------|----------|
| 物理机/VM systemd 正常 | Systemd | systemd |
| 容器内 systemd | Cron | cron + 后台进程 |
| 无 systemd 的 Linux | Cron | cron + 后台进程 |

### 7.3 Systemd 实现

#### 权限说明

生成 systemd unit 文件需要 root 权限，处理方式：

```bash
# 方案 A：提示用户使用 sudo（默认）
# htbox 会检测权限，不足时提示：
# "Permission denied. Try: sudo htbox service enable my-service"

# 方案 B：使用 user-level systemd（无需 root）
# unit 文件放在 ~/.config/systemd/user/
# 命令使用 systemctl --user
```

**默认使用方案 A**。如需 user-level systemd，可在 config.toml 中配置：

```toml
[backend]
  force = "systemd"
  user_level = true  # 使用用户级 systemd
```

#### 生成 unit 文件

```ini
# /etc/systemd/system/htbox-my-service.service
[Unit]
Description=my-service - 数据同步服务
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/user/.htbox/services/my-service
EnvironmentFile=/home/user/.htbox/services/my-service/.env
ExecStart=/home/user/.htbox/services/my-service/wrapper.sh
Restart=on-failure
RestartSec=5
StandardOutput=append:/home/user/.htbox/services/my-service/logs/stdout.log
StandardError=append:/home/user/.htbox/services/my-service/logs/stderr.log

[Install]
WantedBy=multi-user.target
```

> **注意**：ExecStart 使用 wrapper.sh 而非 script.sh，以确保环境变量正确加载。

### 7.4 Cron 降级实现

#### 7.4.1 常驻服务 (daemon)

使用后台运行 + PID 文件管理：

```
~/.htbox/services/<service>/
├── service.toml
├── script.sh
├── .env
├── logs/
│   ├── stdout.log
│   └── stderr.log
└── run/                      # 运行时目录
    ├── pid                   # PID 文件
    └── lock                  # 锁文件
```

**启动流程**：
```bash
# 1. 检查是否已运行（通过 PID 文件）
if [ -f run/pid ]; then
    pid=$(cat run/pid)
    if kill -0 $pid 2>/dev/null; then
        echo "Service already running (PID: $pid)"
        exit 1
    fi
fi

# 2. 创建 run 目录
mkdir -p run

# 3. 加载环境变量
source .env
source ~/.htbox/envs/global.env

# 4. 使用 nohup 后台运行
nohup ./script.sh >> logs/stdout.log 2>> logs/stderr.log &
echo $! > run/pid
```

**停止流程**：
```bash
if [ -f run/pid ]; then
    pid=$(cat run/pid)
    if kill -0 $pid 2>/dev/null; then
        kill $pid
        # 等待进程退出
        for i in {1..10}; do
            if ! kill -0 $pid 2>/dev/null; then
                break
            fi
            sleep 1
        done
        # 强制杀死（如果仍未退出）
        kill -9 $pid 2>/dev/null
    fi
    rm -f run/pid
fi
```

**重启流程**：stop + start

**进程监控（可选）**：
- 使用 inotifywait 监控进程退出，然后自动重启
- 或者依赖 systemd 的 WatchdogFeature（如果支持）

#### 7.4.2 一次性服务 (onetime)

使用 cron @reboot：

```bash
# 添加 @reboot 任务
crontab -l 2>/dev/null | grep -v "htbox.*onetime" > /tmp/crontab.tmp
echo "@reboot /home/user/.htbox/services/my-service/script.sh >> /home/user/.htbox/services/my-service/logs/stdout.log 2>> /home/user/.htbox/services/my-service/logs/stderr.log" >> /tmp/crontab.tmp
crontab /tmp/crontab.tmp
```

**注意**：
- @reboot 在用户登录后执行（取决于 cron 守护进程）
- 如果需要 root 级别的 @reboot，需要 sudo

#### 7.4.3 Cron 后端目录结构

```
~/.htbox/cron/
├── jobs/                    # cron 任务配置
│   ├── onetime-1.job
│   └── onetime-2.job
└── crontab.backup           # 备份原始 crontab
```

## 11 错误处理

| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| 服务不存在 | 返回错误 | "Service 'xxx' not found" |
| 脚本文件不存在 | 阻止启动 | "Script file not found: xxx" |
| 权限不足 | 提示 + 建议 | "Permission denied. Try: sudo htbox ..." |
| 服务已在运行 | 忽略或重启 | "Service already running, use restart" |
| systemd 不可用 | 自动降级 | "systemd unavailable, using cron backend" |
| 配置格式错误 | 解析失败 | "Invalid config: xxx" |

### Rust 错误类型定义

```rust
use thiserror::Error;

#[derive(Error, Debug)]
pub enum Error {
    #[error("Service not found: {0}")]
    ServiceNotFound(String),
    
    #[error("Script file not found: {0}")]
    ScriptNotFound(String),
    
    #[error("Permission denied: {0}. Try: sudo htbox ...")]
    PermissionDenied(String),
    
    #[error("Service already running: {0}")]
    ServiceAlreadyRunning(String),
    
    #[error("systemd unavailable, using cron backend")]
    SystemdUnavailable,
    
    #[error("Invalid config: {0}")]
    ConfigError(String),
    
    #[error("IO error: {0}")]
    IoError(#[from] std::io::Error),
    
    #[error("TOML parse error: {0}")]
    TomlError(#[from] toml::de::Error),
}
```

### 服务删除流程

```bash
# htbox service remove my-service

# 1. 停止服务（如果正在运行）
htbox service stop my-service

# 2. 取消开机自启
htbox service disable my-service

# 3. 清理 systemd unit 文件（如果使用 systemd）
sudo rm -f /etc/systemd/system/htbox-my-service.service
systemctl daemon-reload

# 4. 清理 cron 任务（如果使用 cron 且是 onetime 类型）
crontab -l | grep -v "my-service" | crontab -

# 5. 删除服务目录
rm -rf ~/.htbox/services/my-service

# 6. 更新 state.json
```
